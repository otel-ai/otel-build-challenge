# otel-scaffold-v1 — recommended project layout

This folder is a **starter layout** some teams copy into their solution repository.
It is not required for submission and is **not** part of the official challenge
deliverables.

If you are building manually, you do not need any of this. If you use coding
agents, read the main [README.md](../README.md) phase gates instead of copying
this tree wholesale.

## Suggested directories (optional)

```
your-solution-repo/
├── .implementation-notes/     # scratch notes — do not submit
├── HACKATHON_PROGRESS.md      # agent task log — do not submit
├── AGENTS.md                  # agent instructions — optional
├── implementation/            # rm-agent-bootstrap workspace
│   └── PROGRESS.md
├── etl/
├── tools/
├── skills/
└── tests/
```

Copy this `.otel-scaffold/` folder into your repo root only if you understand
each file and delete anything you do not need before submission.
